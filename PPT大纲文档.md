## 第4页：作品简介

### 大纲内容

**核心简介（200字左右）**

工友通是一款面向家电维修、水电维修、空调安装维修等蓝领工人的AI全链路工作助手应用。它覆盖维修工人从接单、排程、导航、诊断、维修、报价到知识沉淀的完整工作流，通过深度集成9大AI服务，将人工智能嵌入每一个业务节点，帮助维修工人实现接单更高效、维修更专业、记录更规范、经验可沉淀、收入可分析、成长可持续的全方位提升。

**详细说明**

| 维度 | 内容 |
|:---|:---|
| 面向人群 | 家电维修工、水电维修工、空调安装维修工、暖通/电工/管道工、小型维修团队负责人、售后服务工程师 |
| 应用场景 | 维修工人日常工作：接单、排程、导航、现场诊断、维修过程、报价结算、知识沉淀、收入分析 |
| 解决的问题 | 信息处理成本高、维修效率低、记录不规范、收入不透明、经验难以沉淀传承 |
| 突出创新点 | AI深度嵌入工作流、实时视频维修指导、多AI服务集成、智能降级策略 |
| 主要功能 | 智能工作台、AI视频维修、工单管理、智能知识库、客户管理、收入分析、AI日程管理、百度地图导航 |

### 大纲内容

> 将AI深度嵌入维修工作流的每一个关键节点，实现从接单到知识沉淀的全链路智能化，而非简单的工具叠加。

**设计背景与洞察**

| 洞察 | 说明 |
|:---|:---|
| 行业现状 | 蓝领工人数量庞大，但信息化程度低，工作效率低下 |
| 用户痛点 | 维修工人面临多重挑战：接单、排程、维修、记录、收入分析 |
| 市场空白 | 现有工具多为功能堆叠，缺乏工作流整合和AI深度赋能 |
| 交互需求 | 工人双手常被占用，需要低输入成本、语音优先的交互方式 |

**理念贯穿性**

| 设计维度 | 理念体现 |
|:---|:---|
| 功能设计 | 围绕维修工单生命周期构建，AI嵌入每个业务节点 |

| 交互设计 | 语音优先、拍照优先、自动填写，降低输入成本 |
| 技术架构 | 多AI服务协同，智能降级，确保系统可用性 |

## 第11页：交互设计

### 大纲内容

**核心操作流程1：AI视频维修流程**

```
启动维修 → 设备识别 → 故障诊断 → 步骤指导 → 动作验证 → 完工检查
    ↓          ↓          ↓          ↓          ↓          ↓
点击开始   AI自动识别  基于图像   实时语音   AI确认操作  自动生成
CameraX   设备型号    分析判断   引导步骤   是否正确   维修报告
```

**核心操作流程2：智能日程创建流程**

```
语音输入 → AI解析 → 意图判断 → 结构化生成 → 确认创建 → 智能提醒
    ↓         ↓         ↓           ↓           ↓          ↓
说一句话  提取时间   判断是创建   生成结构化   用户确认   AI生成
描述日程  地点信息   日程还是问答  日程数据    后创建     准备清单
```

| 交互方式 | 说明 |
|:---|:---|

| AR叠加层 | 实时视频画面叠加AI指导信息，边做边学 |

**核心代码示例**

```java
// VivoAiService.java - AI意图分类示例
public class VivoAiService {
    public void classifyIntent(String userInput, Callback callback) {
        // 构建prompt，让AI判断用户意图
        String prompt = "判断用户意图是'创建日程'还是'知识问答'：" + userInput;
        
        // 调用蓝心大模型API
        aiApiClient.streamChat(prompt, new StreamCallback() {
            @Override
            public void onStream(String response) {
                // 流式返回分类结果
                callback.onResult(response);
            }
        });
    }
}
```
