# 蓝心大模型图片生成功能修复计划

## 问题分析

根据文档 `蓝心大模型能力\图片生成.docx` 和项目代码对比，发现以下问题：

### 问题 1：响应解析不完整（关键 Bug）

**文档中的响应格式：**
```json
{
    "code": 0,
    "message": "success",
    "data": {
        "image": "https://...",  // 字符串格式的单张图片 URL
        "images": [              // 数组格式的图片列表
            {"url": "https://...", "size": "2048x2048"}
        ]
    }
}
```

**当前代码问题：**
`ImageGenerationResult.java` 第 92 行只解析 `data.images` 数组：
```java
JSONArray imagesArray = data.optJSONArray("images");
```

如果 API 只返回 `data.image`（字符串）而没有 `data.images`（数组），解析会失败。

**修复方案：**
在 `fromJson()` 方法中添加对 `data.image` 字段的处理作为回退：
```java
// 先尝试解析 data.images 数组
JSONArray imagesArray = data.optJSONArray("images");
if (imagesArray != null && imagesArray.length() > 0) {
    // ... 现有逻辑
} else {
    // 回退：尝试解析 data.image 字符串
    String imageUrl = data.optString("image", "");
    if (!imageUrl.isEmpty()) {
        GeneratedImage generatedImage = new GeneratedImage();
        generatedImage.setUrl(imageUrl);
        imageList.add(generatedImage);
    }
}
```

### 问题 2：使用了文档中不存在的参数

**当前代码（ImageGenerationService.java 第 155 行）：**
```java
parameters.put("sequential_image_generation", "disabled");
```

**文档中的 parameters 示例：**
- `size`: 图片尺寸（如 "2K", "2048x2048"）
- `prompt_extend`: 是否扩展提示词
- `watermark`: 是否添加水印

`sequential_image_generation` 在文档中未提及，可能导致 API 报错。

**修复方案：**
移除 `sequential_image_generation` 参数，只保留 `size`。

### 问题 3：URL 参数可能不必要

**当前代码在 URL 中添加：**
- `module=aigc`
- `request_id={uuid}`
- `system_time={unix_seconds}`

**文档中未明确说明这些参数是否必需。** 虽然添加这些参数可能不会导致错误，但最好确认是否必要。

**修复方案：**
暂时保留这些参数（它们可能用于追踪和限流），但添加注释说明其用途。
