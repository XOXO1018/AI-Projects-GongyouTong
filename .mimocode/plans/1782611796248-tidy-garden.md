# 工友通 — AIGC创新赛参赛完善计划

> **参赛方向**：AIGC+垂直行业（用AI赋能蓝领工人）
> **时间预算**：1周以上
> **材料准备**：用户自行准备

---

## 一、项目核心优势（参赛亮点）

### 1. AIGC技术全覆盖
- **8类AI接口集成**：聊天补全、流式对话、多模态诊断、视觉分析、图片生成、视频生成、OCR、语音交互
- **vivo蓝心大模型深度应用**：Volc-DeepSeek-V3.2 + Doubao-Seedream-4.5
- **完整的AIGC闭环**：从输入（语音/图片）→ AI处理 → 输出（诊断/指导/视频）

### 2. 垂直行业应用价值
- **目标用户明确**：家政维修工人（蓝领群体数字化）
- **痛点解决真实**：维修经验不足、诊断困难、安全隐患
- **社会效益显著**：提升蓝领技能、降低维修事故、促进行业标准化

### 3. 技术创新亮点
- **双通道帧分析架构**：JoyAI-VL主动式指导 + 本地Vision降级
- **7状态维修状态机**：严密的状态流转 + 灵活跳转
- **语音全链路闭环**：ASR → 意图识别 → 操作分派 → TTS反馈
- **AR叠加标注**：实时视频 + 归一化坐标 + 警告横幅
- **三级降级策略**：云端AI → 本地fallback → 默认规则

## 二、参赛重点方向（AIGC+垂直行业）

**核心叙事**：用AIGC技术赋能蓝领工人，让AI成为每个维修工人的"智能师傅"

**展示重点**（按演示顺序）：

### 2.1 智能日程管理（开场）
- 用户说"明天下午3点去幸福小区修空调"
- AI自动解析：标题、地址、设备类型、时间
- 生成出发前准备清单（工具、材料、安全事项）

### 2.2 多模态故障诊断（核心）
- 拍照 + 语音描述故障
- AI多模态分析：故障原因、排查步骤、所需工具配件
- 结构化诊断报告展示

### 2.3 AI维修步骤规划
- 根据诊断结果生成维修步骤
- 每步包含：操作说明、工具要求、安全提示
- 支持语音逐步确认

### 2.4 AR实时指导（亮点）
- 视频画面 + AI视觉分析
- AR叠加：高亮框标注操作区域
- 安全警告横幅（红色闪烁）
- 错误检测与纠正

### 2.5 知识库管理（闭环）
- OCR识别设备铭牌
- AI生成知识摘要和思维导图
- 维修经验沉淀复用

### 2.6 AI报价生成
- 根据维修内容自动生成报价单
- 配件+人工+其他费用明细

## 三、需要完善的功能模块

### 优先级1：核心功能修复（必做）

#### 1.1 OCR API修复
- **问题**：vivo OCR API `/api/v1/ocr` 返回404
- **方案**：已改为使用AI多模态聊天接口实现OCR
- **验证**：测试图片识别功能是否正常

#### 1.2 知识库OCR+AI总结流程
- **当前状态**：OCR调用已集成，但需要端到端测试
- **需要完善**：
  - 选择图片 → OCR识别 → 填入输入框 → 保存 → AI生成摘要+思维导图
  - 测试完整流程是否顺畅

#### 1.3 AI诊断功能验证
- **需要测试**：
  - 上传故障图片 → 多模态诊断 → 结构化结果展示
  - 诊断结果解析是否正确（DiagnosisResult字段）

### 优先级2：演示效果优化（重要）

#### 2.1 预置演示数据
- **知识库**：预置5-10条常见维修知识（空调、洗衣机、冰箱等）
- **工单**：预置3-5个典型故障工单
- **诊断结果**：缓存典型诊断结果，避免现场AI响应慢

#### 2.2 UI/UX优化
- **加载状态**：AI处理时显示进度提示（"AI正在分析..."）
- **错误处理**：AI失败时的友好提示和重试机制
- **动画效果**：AR叠加的平滑过渡

#### 2.3 性能优化
- **图片压缩**：上传前压缩到合适大小（减少AI处理时间）
- **缓存机制**：常用诊断结果缓存
- **超时处理**：设置合理超时，避免长时间等待

## 五、风险点与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| AI响应慢 | 演示卡顿 | 预置缓存数据 + 图片压缩 |
| OCR识别失败 | 功能不可用 | 已改为多模态方案 + 手动输入降级 |
| 网络不稳定 | AI调用失败 | 三级降级策略 + 离线fallback |
| JoyAI-VL不可用 | AR指导失效 | 本地Vision分析降级 |
